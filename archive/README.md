# 归档目录说明

本目录用于存放项目开发过程中产生的临时性、过时或已被替代的文件。

## 归档规则

1. **测试脚本归档** (`archive/tests/`)
   - 存放过时或已被新版本替代的测试脚本
   - 按功能分类存放
   - 保留日期信息便于追溯

2. **文档归档** (`archive/docs/`)
   - 存放过时、重复或被替代的总结文档
   - 保留历史记录用于审计和参考
   - 按日期或功能分类

## 归档日期

归档创建于: 2026-01-26

## 归档原因

项目经过多次迭代和功能修复，产生了大量临时测试文件和总结文档。为保持项目根目录的整洁，将这些文件归档到本目录。

## 注意事项

- 归档的文件仅供参考，不应直接用于生产环境
- 如需恢复某个文件，请先评估其当前适用性
- 建议定期清理归档目录，删除不再需要的文件
